# Self-Check System

The self-check is not a summary. It is a publish gate. If any gate fails, revise the article before handoff.

## Three Success Standards

### 1. Real Evidence Fully Matches The Copy

Pass only if every important claim can answer:

1. Which source proves it?
2. Which screenshot or recording supports the nearby sentence?
3. Does the caption say exactly what the image proves, without overstating?

Prefer real recordings when they prove workflow, comparison, product behavior, or a visible result. Use screenshots for official statements, data tables, product pages, quotes, and fixed facts.

### 2. Zero AI Flavor Plus Strong First Screen

Run a hard scan for:

- empty industry language
- mechanical transitions
- "firstly / secondly / finally" structure
- generic "this means" summary
- fake balanced tone
- backend/editor notes
- over-neat bullet logic
- repeated "user" wording in Chinese copy

Then inject the user's own voice:

- first-person reaction
- "我们 / 咱们" when talking about shared fate
- concrete domestic-user constraints such as account, region, identity, organization permission
- natural spoken endings where useful
- ending with emotional complexity and a real exit

Use a 50-point scan:

| Dimension | Pass Target |
|---|---|
| Directness | Says the point without ceremonial setup |
| Rhythm | Long and short sentences vary naturally |
| Trust | Evidence and boundaries are clear |
| Human Voice | Sounds like the user made a judgment |
| Precision | No filler, no exaggerated promise |

Below 45/50 means revise.

### 3. About 2500-3000 Chinese Characters For Hot News

For regular AI hot-news pieces, write enough to explain the material, usually 2500-3000 Chinese characters and no more than about 3000 unless the topic is a deep essay. Do not stretch with filler. Do not compress user-provided core material into a thin outline.

## Reader Six-Question Gate

Read as the target WeChat reader and answer each with "yes / no / weak" plus one reason:

1. Does this help me with a real problem?
2. Is there real useful content, not just correct-sounding commentary?
3. Is the reading experience smooth on mobile?
4. Would I follow this account after reading?
5. Would I share or save it?
6. Is there one sentence I would quote, like, or remember?

Any "no" or "weak" sends the draft back to the relevant loop:

- evidence or usefulness problem -> evidence and content loop
- AI-flavor or awkward rhythm -> voice loop
- first-screen problem -> title, opening, and first visual loop
- no share/save moment -> reader-action loop

After revising, rerun all six questions, not only the failed one.

## HTML And Artifact Gate

Before final handoff:

- Markdown, HTML, and visible browser copy must match.
- Image count must match planned evidence count.
- Every image must have a Chinese caption below it.
- No old material-folder path appears in the new article.
- Copy button and JS syntax work if the page has JavaScript.
- Local images remain available as fallback even when remote links are used.
- The final response includes the exact paths and remaining manual publishing step.
